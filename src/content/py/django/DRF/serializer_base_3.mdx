
## 📚 Django REST Framework 序列化器 (Serializers) 全能笔记

### 一、 核心职责：数据的双向翻译 🔄
序列化器处于 Django 模型（Model）和客户端（Frontend）之间，负责两种转换：
1.  **序列化 (Serialization)**：把复杂的 `Model 实例` 或 `QuerySet` 变成 Python 原生数据（通常是字典），随后被渲染成 **JSON**。
2.  **反序列化 (Deserialization)**：将客户端发来的 **JSON** 数据还原成 Python 字典，并最终保存为 `Model 实例`。

### 二、 安全防线：验证机制 (Validation) 🛡️
在数据存入数据库前，必须经过“安检”。
* **动作**：必须调用 `is_valid()`。
* **结果**：
    * 成功：数据进入 `validated_data` 字典。
    * 失败：错误信息存入 `.errors`（可配合 `raise_exception=True` 自动返回 400 错误）。
* **自定义规则**：
    * **字段级 (`validate_字段名`)**：针对单个字段（如：检查内容是否有敏感词）。
    * **对象级 (`validate`)**：针对多个字段的逻辑（如：检查结束日期是否晚于开始日期）。

### 三、 生产力工具：ModelSerializer 🪄
它像是一个“自动模板”，通过 `Meta` 类直接与数据库模型绑定：
* **model**: 指定绑定的 Django 模型。
* **fields**: 指定要在 API 中展示的字段（`['id', 'title']` 或 `'__all__'`）。
* **read_only_fields**: 保护字段不被前端修改（如 `id`, `created_at`, `author`）。

### 四、 复杂关系：嵌套序列化 (Nested) 🪆
处理一对多或多对多关系。例如在“文章”中展示“评论列表”：
* **关键参数**：`many=True`（告诉 DRF 这是一个列表）。
* **展现形式**：JSON 会呈现出完美的层级结构。

### 五、 协同作战：保存逻辑的分工 🤝
这是理解 DRF 架构的关键：
1.  **Serializer.create() (后厨)**：只负责“怎么写进数据库”。
2.  **View.perform_create() (经理)**：负责“补充额外信息”。它从 `request` 中获取信息（如当前登录用户），并通过 `serializer.save(author=user)` 传给后厨。

---

## 💻 综合实战代码示例

```python
# serializers.py
class ArticleSerializer(serializers.ModelSerializer):
    # 嵌套评论
    comments = CommentSerializer(many=True, read_only=True)

    class Meta:
        model = Article
        fields = ['id', 'title', 'content', 'author', 'comments']
        read_only_fields = ['id', 'author'] # 防止前端篡改作者和ID

    def validate_title(self, value):
        if len(value) < 5:
            raise serializers.ValidationError("标题太短了！")
        return value

# views.py
class ArticleViewSet(viewsets.ModelViewSet):
    def perform_create(self, serializer):
        # 联动：将请求用户自动关联为文章作者
        serializer.save(author=self.request.user)
```
