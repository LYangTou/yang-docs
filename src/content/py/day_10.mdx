# Day 10 笔记总结：Python 面向对象（OOP）基础

## 1）类 vs 对象（实例）

* **类（class）**：模板/蓝图，定义“有哪些属性、有哪些行为（方法）”
* **对象/实例（instance）**：由类创建出来的具体实体

  ```python
  s = Student("Tom", 18)  # s 是 Student 的实例
  ```

---

## 2）`__init__` 构造函数与 `self`

* `__init__`：创建对象时自动调用，用来**初始化属性**
* `self`：表示“当前这个实例”，方法里通过 `self.xxx` 访问/修改实例数据

  ```python
  class Student:
      def __init__(self, name):
          self.name = name
  ```

✅ 记住：**定义方法时必须写 self（第一个参数）**；调用方法时不用传 self（Python 自动传）

---

## 3）属性：实例变量 vs 类变量（高频易错）

### 实例变量（每个对象一份）

* 通常写在 `__init__` 里：`self.score = score`

### 类变量（所有对象共享一份）

```python
class Student:
    school = "No.1"  # 类变量（共享）
```

⚠️ 易错点：把想做“每个对象不同”的值写成了类变量，会导致所有实例共享同一份数据。

---

## 4）方法（行为）

* 实例方法：最常用，定义/调用方式：

  ```python
  class A:
      def run(self):
          ...
  a = A()
  a.run()
  ```

---

## 5）封装（Encapsulation）

核心思想：

* 把**数据（属性）**和**操作数据的逻辑（方法）**放在一起
* 对外暴露清晰接口（方法），内部实现细节可随时调整

例：用 `update_score()` 控制成绩更新，而不是外部随便改。

---

## 6）继承（Inheritance）

* 子类继承父类的属性和方法：

  ```python
  class Dog(Animal):
      ...
  ```
* 子类可以**复用父类能力**，并可扩展新能力
* 若子类重写了 `__init__`，常需要：

  ```python
  super().__init__(...)
  ```

---

## 7）多态（Polymorphism）与方法重写

* 子类可以**重写**父类同名方法
* 调用时会根据实际对象类型执行对应版本（“同一个接口，不同实现”）

---

## 8）`__str__`（非常实用）

* 定义对象被 `print()` 时的展示形式：

  ```python
  def __str__(self):
      return f"Student(name={self.name})"
  ```

✅ 后续调试、日志输出非常方便

---

## 9）今天的实战意义（你做的两个练习能迁移到后端）

* **Student 类**：练习“属性 + 方法 + 状态更新”
* **ConfigLoader 类**：练习“封装 + 容错 + 对外接口（get/load）”
  后端/AI 项目里“读配置、校验参数、默认值回退”非常常见


---

# 常见坑（避免）

* ❌ 方法定义漏写 `self`
* ❌ 在方法里写 `name` 而不是 `self.name`
* ❌ 把实例变量写成类变量导致共享
* ❌ 子类重写 `__init__` 却没 `super().__init__()`（导致父类属性没初始化）
