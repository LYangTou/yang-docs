# Day 7 笔记：模块（module）与包（package）

## 1）模块是什么

* **模块 = 一个 `.py` 文件**
* 文件名（去掉 `.py`）就是模块名
  例：`mymath.py` → 模块名 `mymath`

模块的作用：

* **复用代码**（函数/类/变量放进去，别的地方 import 用）
* **隔离命名空间**：不同模块里同名变量/函数不会互相冲突

---

## 2）包是什么

* **包 = 一个目录 + 里面有 `__init__.py`**
* 有了 `__init__.py` Python 才会把这个目录识别为包

例：

```
mypkg/
  __init__.py
  modA.py
  modB.py
```

* `mypkg` 是包
* `mypkg.modA`、`mypkg.modB` 是包里的模块

---

## 3）import 的三种常用写法

### 3.1 `import xxx`

```python
import mymath
mymath.square(3)
```

* 导入整个模块
* 使用时必须写 `模块名.成员`

✅ 优点：不容易命名冲突，推荐常用

---

### 3.2 `from xxx import yyy`

```python
from mymath import square
square(3)
```

* 只导入某个成员
* 使用时不用写模块前缀

⚠️ 缺点：容易跟别的函数同名冲突（比如 `square` 你自己也写了一个）

---

### 3.3 别名 `as`

```python
import numpy as np
```

* 常用在名字长的模块上，或者减少输入

---

## 4）`__name__` 与入口保护（必会）

在每个模块里都有一个变量：`__name__`

* **直接运行这个文件**：`__name__ == "__main__"`
* **被别人 import**：`__name__ == "模块名"`

所以常见写法：

```python
def main():
    ...

if __name__ == "__main__":
    main()
```

作用：

* ✅ 允许模块“既能被导入复用”，又能“直接运行做测试”
* import 时不会执行 main 里的测试代码

---

## 5）模块搜索路径（了解即可）

Python import 时会按顺序找模块：

* 当前运行脚本目录
* 环境变量、站点包目录（第三方库）
* Python 标准库目录

⚠️ 常见坑：不要把自己的文件命名成标准库名字
比如你写 `random.py`，会覆盖标准库 `random`

---

## 6）包内导入：绝对导入 vs 相对导入

### 6.1 绝对导入（推荐）

```python
from mypkg.modA import foo
```

* 写全路径，清晰稳定

### 6.2 相对导入（包内部才用）

```python
from .modA import foo
```

* `.` 表示当前包
* ⚠️ 不能在“直接运行的主脚本”里用相对导入
  （因为主脚本没有包上下文）
