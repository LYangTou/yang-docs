## 抓住三大集合 List / Set / Map

集合可以认为是对数据进行**增删改查**的容器。  
集合框架是用于标识和操作集合的统一体系结构（framework）。

由三个主要部分组成：
- **接口**：标识集合的抽象
- **实现**：集合接口的具体实现。本质上，它们是可重用的数据结构
- **算法**：对实现集合接口的对象执行计算的方法。相同的方法可以用于适当集合接口的许多不同实现（多态）。本质上，算法是可重用的功能。

**集合代码的相关位置**
- 大部分集合类位于 `java.util` 包下
- 在 `java.util.concurrent` 包下提供了**并发容器**（如 `ConcurrentHashMap`、`CopyOnWriteArrayList`）

## 三大重要集合接口和实现类
- `List` → `ArrayList`（动态数组）
- `Set` → `HashSet`（高中数学中的集合，无重复元素）
- `Map` → `HashMap`（字典，键值对）

```
          | Collection 接口 |                     | Map 接口 |
                |                                   |
                |                                   |
        List 接口 -- | -- Set 接口                HashMap
            |                |
            |                |
        ArrayList         HashSet
```

### Collection
集合的根 `interface`，具体的方法由各自的子类去实现。  
提供两个**标准构造器**：一个无参构造器，一个参数为 `Collection` 类型的拷贝构造器。

**重要约定**：  
集合框架中的很多操作（如 `contains`、`remove`）都依赖元素的 `equals` 方法来判断对象是否相等。  
如果自定义类作为集合元素，**重写 `equals` 就必须重写 `hashCode`**，否则在 `HashSet` / `HashMap` 中会表现异常。

`Collection` 接口实际上继承自 `Iterable` 接口，因此所有 `Collection` 的子类都支持 `for-each` 遍历。

#### 暴露的重要方法

**Query Operations（查询操作）**
- `size()`
- `isEmpty()`
- `contains(Object o)`
- `iterator()`：返回迭代器
- `toArray()`：返回一个同顺序的 `Object` 数组

**Modification Operations（修改操作）**
- `add(E e)`
- `remove(Object o)`

**Bulk Operations（批量操作）**
- `containsAll(Collection<?> c)`
- `addAll(Collection<? extends E> c)`
- `removeAll(Collection<?> c)`
- `clear()`

### Iterable
实现该接口的对象（即继承该接口的类）就允许使用 `for-each` 进行遍历。所以所有 `Collection` 集合的对象都可以使用 `for-each`。

- `iterator()`：返回一个迭代器
- `forEach(Consumer<? super T> action)`：JDK 8 引入，支持 Lambda 遍历
- `spliterator()`：用于并行遍历的分割迭代器

### Iterator
代替了 `Enumeration`（已废弃），允许从底层集合中**安全删除元素**，让集合可以动态变化。

- `hasNext()`：判断是否有更多元素
- `next()`：获取下一个元素
- `remove()`：**可选操作**。大多数集合的迭代器会正确实现它（如 `ArrayList.iterator()` 实现的 `remove` 会删除当前元素）。但如果迭代器不支持删除，调用会抛 `UnsupportedOperationException`。
- `forEachRemaining(Consumer<? super E> action)`：对剩余元素执行操作

**迭代器实现例子**（你的例子很好，但注意泛型与 remove 的可选性）：
```java
public class MyIterator implements Iterator<Integer> {
    private final int[] nums;
    private int cursor;
    public MyIterator() {
        this.nums = new int[]{1, 4, 3, 6, 7, 9};
        this.cursor = 0;
    }

    @Override
    public boolean hasNext() {
        return cursor != nums.length;
    }

    @Override
    public Integer next() {
        return nums[cursor++];
    }

    // 不实现 remove() 则默认抛异常，符合可选操作规范

    public static void main(String[] args) {
        Iterator<Integer> i = new MyIterator();
        while (i.hasNext()) {
            int num = i.next();
            System.out.println(num);
        }
    }
}
```

### List
是有序集合，和数组很像。可以通过**下标**或者**搜索元素**来访问。**允许重复元素**。

**特有方法**（除了继承自 `Collection` 的）：
- `sort(Comparator<? super E> c)`：排序
- `addAll(int index, Collection<? extends E> c)`：从指定位置插入
- `get(int index)` / `set(int index, E element)`：按索引读写
- `indexOf(Object o)` / `lastIndexOf(Object o)`
- `remove(int index)`：按索引删除
- `subList(int fromIndex, int toIndex)`：返回视图（非独立副本）

**常用实现类**：
- `ArrayList`：动态数组，随机访问快，中间插入/删除慢
- `LinkedList`：双向链表，中间插入/删除快，随机访问慢（也实现了 `Deque`）

内部有ListIterator

### Queue
继承自 `Collection`，JDK 1.8 可以实现**队列**和**栈**。

**主要方法对比**（每组两种行为，失败时异常 vs 返回特殊值）：

| 操作 | 抛出异常    | 返回特殊值   |
|------|------------|--------------|
| 插入 | `add(e)`   | `offer(e)`   |
| 删除 | `remove()` | `poll()`     |
| 查看 | `element()`| `peek()`     |

**双端队列** `Deque`（发音 "deck"）：
- 支持在两端插入/删除：`addFirst` / `addLast`，`offerFirst` / `offerLast`
- 可作为栈使用：`push(e)` 等效 `addFirst(e)`，`pop()` 等效 `removeFirst()`

### Set
不允许重复元素（通过 `equals` 和 `hashCode` 判断）。**无索引**，元素无序（`HashSet`）或可排序（`TreeSet`）。

**常用实现类**：
- `HashSet`：基于 `HashMap` 实现，无序，允许一个 `null`
- `LinkedHashSet`：维护插入顺序，性能略低于 `HashSet`
- `TreeSet`：基于 `TreeMap` 实现，元素**可排序**（要求元素实现 `Comparable` 或传入 `Comparator`），不允许 `null`（JDK 8+ 中若 Comparator 允许 null 则例外）

**重要方法**（来自 `Collection`，无额外索引方法）：
- `add(E e)`：如果已存在则返回 `false`
- 批量操作：`addAll` 等

### Map
**键值对**容器，一个键对应一个值。键不可重复（通过 `equals` 判断）。  
**常用实现类**：
- `HashMap`：无序，允许一个 `null` 键和多个 `null` 值，默认初始容量 16，负载因子 0.75
- `LinkedHashMap`：维护插入顺序或访问顺序（可用作简易 LRU 缓存）
- `TreeMap`：按键的自然顺序或 `Comparator` 排序，不允许 `null` 键（值可以）
- `Hashtable`：遗留类，线程安全但不推荐，不允许 `null`
- `ConcurrentHashMap`：并发容器，高并发下性能好，不允许 `null`

**标准构造器**（同 `Collection` 的建议）：
- 无参构造器
- 带一个 `Map` 类型参数的拷贝构造器

**核心约定**：同一个 Map 里，任意两个键的 `equals` 永远不相等。  
Map 内部通过 `hashCode` 定位桶，再通过 `equals` 确认。

#### 内部接口 `Entry<K,V>`
提供：
- `getKey()`
- `getValue()`
- `setValue(V value)`
- `equals(Object o)`
- `hashCode()`

#### 重要方法

**Basic Operations**
- `size()`
- `isEmpty()`
- `containsKey(Object key)`
- `containsValue(Object value)`
- `get(Object key)`

**Modification Operations**
- `put(K key, V value)`：已有 key 则替换 value，否则新增
- `remove(Object key)`

**Bulk Operations**
- `putAll(Map<? extends K, ? extends V> m)`
- `clear()`

**Collection Views**（返回视图，改变视图会反映回原 Map）
- `keySet()`：返回所有键的 `Set`
- `values()`：返回所有值的 `Collection`（允许重复值）
- `entrySet()`：返回所有 `Entry` 的 `Set`

**Default Methods (JDK 8+)**
- `getOrDefault(Object key, V defaultValue)`：key 不存在则返回默认值
- `putIfAbsent(K key, V value)`：仅当 key 不存在或值为 null 时插入
- `remove(Object key, Object value)`：仅当键值对匹配时才删除
- `replace(K key, V oldValue, V newValue)`：条件替换
- `replace(K key, V value)`：无条件替换（仅当 key 存在）
- `compute`、`computeIfAbsent`、`computeIfPresent`、`merge` 等函数式方法

## 补充：集合遍历的快速失败机制（Fail-Fast）

当使用迭代器遍历集合时，如果**在迭代过程中**通过集合自身的修改方法（非迭代器的 `remove`）改变了集合结构，迭代器会立刻抛出 `ConcurrentModificationException`。  
这可以防止不可预期的并发修改。`ArrayList`、`HashSet`、`HashMap` 等都采用了快速失败机制。

**例外**：`ConcurrentHashMap`、`CopyOnWriteArrayList` 等并发容器使用**弱一致性**迭代器，不会抛此异常。

## 补充：Collections 工具类

`java.util.Collections` 提供了大量静态方法用于操作集合：
- 排序：`sort(List<T> list)`
- 打乱：`shuffle(List<?> list)`
- 反转：`reverse(List<?> list)`
- 不可修改视图：`unmodifiableList`、`unmodifiableMap` 等
- 同步包装器：`synchronizedList`、`synchronizedMap` 等（注意：并发性低于 `ConcurrentHashMap`）
- 单元素集合：`singletonList`、`singletonMap`
- 空集合：`emptyList`、`emptyMap`、`emptySet`

## 补充：并发容器（java.util.concurrent）

- `ConcurrentHashMap`：高并发下替代 `Hashtable`，分段锁（JDK 7）/ CAS + synchronized（JDK 8+）
- `CopyOnWriteArrayList`：读多写少场景，写时复制底层数组
- `BlockingQueue` 接口及其实现（`ArrayBlockingQueue`、`LinkedBlockingQueue` 等）：支持阻塞的队列操作
