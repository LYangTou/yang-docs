# Java 集合：List / Set / Map 笔记

---

## 一、什么是集合

集合可以理解为：

* 用来存放多个数据的容器
* 主要用于对数据进行**增删改查**

和数组相比，集合更灵活：

* 数组长度固定
* 集合长度通常可变

---

## 二、集合框架的组成

Java 集合框架主要由三部分组成：

### 1. 接口

定义集合的规范，例如：

* `Collection`
* `List`
* `Set`
* `Map`

### 2. 实现类

接口的具体实现，本质上就是可复用的数据结构，例如：

* `ArrayList`
* `HashSet`
* `HashMap`

### 3. 算法 / 工具方法

用于操作集合的一些通用方法，例如排序、打乱、反转等，常见于：

* `Collections`
* `Arrays`

---

## 三、集合相关包

### 1. 常用集合类

大部分在：

```java
java.util
```

### 2. 并发容器

并发场景下常用的容器在：

```java
java.util.concurrent
```

例如：

* `ConcurrentHashMap`
* `CopyOnWriteArrayList`

---

## 四、三大重要集合接口

最常用的三大类集合：

* `List`
* `Set`
* `Map`

对应常见实现：

* `List` → `ArrayList`
* `Set` → `HashSet`
* `Map` → `HashMap`

---

## 五、集合体系简单理解

### 1. Collection

`Collection` 是单列集合的根接口。

它下面主要有两大子接口：

* `List`
* `Set`

### 2. Map

`Map` 是双列集合接口，和 `Collection` **不是继承关系**。

这个点要记住：

* `List`、`Set` 属于 `Collection`
* `Map` 单独是一套体系

---

## 六、Collection

### 1. 定义

`Collection` 是单列集合的根接口，表示一组对象。

它的子类通常都支持：

* 添加元素
* 删除元素
* 查询元素
* 遍历元素

---

### 2. Collection 常用方法

#### 查询操作

```java
size()
isEmpty()
contains(Object o)
iterator()
toArray()
```

#### 修改操作

```java
add(E e)
remove(Object o)
```

#### 批量操作

```java
containsAll(Collection<?> c)
addAll(Collection<? extends E> c)
removeAll(Collection<?> c)
clear()
```

---

### 3. 重要说明

`Collection` 继承了 `Iterable`，所以所有 `Collection` 子类都可以使用：

```java
for-each
```

例如：

```java
for (String s : list) {
    System.out.println(s);
}
```

---

## 七、Iterable

### 1. 作用

实现了 `Iterable` 的对象，就可以使用增强 `for` 遍历。

### 2. 常用方法

```java
iterator()
forEach(Consumer<? super T> action)
spliterator()
```

基础阶段最重要的是记住：

* `iterator()`
* 支持 `for-each`

---

## 八、Iterator

### 1. 作用

迭代器用于遍历集合元素。

### 2. 常用方法

```java
hasNext()
next()
remove()
```

---

### 3. 示例

```java
Iterator<String> it = list.iterator();

while (it.hasNext()) {
    String s = it.next();
    System.out.println(s);
}
```

---

### 4. remove() 的注意点

`Iterator` 的 `remove()` 是删除**当前迭代到的元素**。
它是安全删除集合元素的常用方式。

```java
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if ("a".equals(s)) {
        it.remove();
    }
}
```

---

## 九、List

### 1. 特点

`List` 是有序集合，特点：

* 有顺序
* 允许重复元素
* 可以通过索引访问

---

### 2. 常用方法

除了继承自 `Collection` 的方法，还常用这些：

```java
get(int index)
set(int index, E element)
add(int index, E element)
remove(int index)
indexOf(Object o)
lastIndexOf(Object o)
subList(int fromIndex, int toIndex)
sort(Comparator<? super E> c)
```

---

### 3. List 常见实现类

#### ArrayList

底层是动态数组。

特点：

* 查询快
* 随机访问快
* 中间插入、删除较慢

最常用。

---

#### LinkedList

底层是双向链表。

特点：

* 中间插入、删除相对快
* 随机访问慢

同时它还实现了：

* `Deque`

所以也可以当队列、双端队列用。

---

### 4. 什么时候用 List

当你需要：

* 保存元素顺序
* 允许重复
* 可能通过索引访问

通常优先考虑 `List`。

---

## 十、Queue

### 1. 定义

`Queue` 表示队列，通常遵循：

* 先进先出（FIFO）

---

### 2. 常用方法对比

| 操作   | 抛异常         | 返回特殊值      |
| ---- | ----------- | ---------- |
| 插入   | `add(e)`    | `offer(e)` |
| 删除   | `remove()`  | `poll()`   |
| 查看队头 | `element()` | `peek()`   |

---

### 3. 区别

核心区别在于：**失败时的处理方式不同**

* `add/remove/element`：失败时抛异常
* `offer/poll/peek`：失败时返回特殊值

例如：

* `poll()` 失败返回 `null`
* `peek()` 失败返回 `null`

---

## 十一、Deque

### 1. 定义

`Deque` 表示双端队列，可以从两端操作元素。

### 2. 常用方法

```java
addFirst()
addLast()
offerFirst()
offerLast()
removeFirst()
removeLast()
peekFirst()
peekLast()
```

---

### 3. 作为栈使用

`Deque` 也可以当栈：

```java
push(e)
pop()
peek()
```

一般现在更推荐用 `Deque` 实现栈，而不是用老的 `Stack` 类。

---

## 十二、Set

### 1. 特点

`Set` 表示不允许重复元素的集合。

特点：

* 不允许重复
* 没有索引
* 一般不能通过下标访问

---

### 2. 重复的判断依据

`Set` 判断元素是否重复，通常依赖：

* `equals()`
* `hashCode()`

所以如果自定义对象要放进 `HashSet`，通常要重写：

* `equals()`
* `hashCode()`

---

### 3. Set 常见实现类

#### HashSet

最常用。

特点：

* 无序
* 不允许重复
* 允许一个 `null`

底层基于 `HashMap`。

---

#### LinkedHashSet

特点：

* 不允许重复
* 维护插入顺序

比 `HashSet` 多了顺序特性。

---

#### TreeSet

特点：

* 不允许重复
* 可以排序

要求元素：

* 实现 `Comparable`
* 或创建时传入 `Comparator`

一般不允许 `null`。

---

### 4. 什么时候用 Set

当你需要：

* 元素不能重复
* 不关心索引
* 可能需要去重

优先考虑 `Set`。

---

## 十三、Map

### 1. 定义

`Map` 是键值对容器，存储形式是：

```java
key -> value
```

特点：

* 键不能重复
* 值可以重复

---

### 2. Map 常见实现类

#### HashMap

最常用。

特点：

* 无序
* 允许一个 `null` 键
* 允许多个 `null` 值

---

#### LinkedHashMap

特点：

* 维护插入顺序
* 也可以维护访问顺序

常用于简易 LRU 场景。

---

#### TreeMap

特点：

* 按键排序
* 键一般不能为 `null`

---

#### Hashtable

老的线程安全类，现在一般不推荐。

特点：

* 线程安全
* 不允许 `null` 键和值

---

#### ConcurrentHashMap

并发场景下常用。

特点：

* 线程安全
* 性能通常比 `Hashtable` 更好
* 不允许 `null`

---

### 3. Map 常用方法

#### 基本操作

```java
size()
isEmpty()
containsKey(Object key)
containsValue(Object value)
get(Object key)
```

#### 修改操作

```java
put(K key, V value)
remove(Object key)
clear()
putAll(Map<? extends K, ? extends V> m)
```

---

### 4. 视图方法

Map 提供三个非常重要的视图：

```java
keySet()
values()
entrySet()
```

#### keySet()

返回所有键的集合。

#### values()

返回所有值的集合。

#### entrySet()

返回所有键值对对象的集合，遍历 Map 时非常常用。

---

### 5. Map 遍历方式

#### 方式1：遍历 keySet

```java
for (String key : map.keySet()) {
    System.out.println(key + " = " + map.get(key));
}
```

#### 方式2：遍历 entrySet（更常用）

```java
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " = " + entry.getValue());
}
```

---

### 6. JDK 8+ 常用默认方法

```java
getOrDefault()
putIfAbsent()
remove(key, value)
replace()
computeIfAbsent()
computeIfPresent()
merge()
```

基础阶段先重点知道：

* `getOrDefault`
* `putIfAbsent`

## 十五、equals 和 hashCode 在集合中的作用

这是集合里非常重要的点。

### 1. List

`List` 可以重复，所以一般不靠 `hashCode` 去重。

### 2. Set / Map

像：

* `HashSet`
* `HashMap`

这类基于哈希的集合，判断重复或定位元素时会用到：

* `hashCode()`
* `equals()`

---

### 3. 结论

如果自定义类作为：

* `HashSet` 元素
* `HashMap` 的键

通常必须重写：

* `equals()`
* `hashCode()`

否则去重、查找可能出现异常表现。

---

## 十六、快速失败机制（Fail-Fast）

### 1. 定义

在使用迭代器遍历集合时，如果遍历过程中直接通过集合本身修改结构，可能会抛出：

```java
ConcurrentModificationException
```

---

### 2. 示例

```java
for (String s : list) {
    if ("a".equals(s)) {
        list.remove(s); // 可能抛异常
    }
}
```

---

### 3. 正确做法

使用迭代器自己的 `remove()`：

```java
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    if ("a".equals(s)) {
        it.remove();
    }
}
```

---

### 4. 注意

并发容器如：

* `ConcurrentHashMap`
* `CopyOnWriteArrayList`

一般不会走这种典型的快速失败机制。

## 十九、这一部分最该记住的点

### 集合体系

* `Collection` 管单列集合
* `Map` 管双列集合
* `List`、`Set` 都属于 `Collection`

### List

* 有序
* 可重复
* 有索引
* 常用：`ArrayList`

### Set

* 不可重复
* 无索引
* 常用：`HashSet`

### Map

* 键值对
* 键不可重复
* 常用：`HashMap`

### Queue / Deque

* `Queue`：队列
* `Deque`：双端队列，也可当栈

### 遍历

* `Collection` 支持 `for-each`
* 删除元素时，优先用迭代器的 `remove()`

### 哈希集合重点

* `HashSet`、`HashMap` 依赖 `hashCode()` 和 `equals()`
* 自定义对象做键或去重元素时，通常要同时重写这两个方法
