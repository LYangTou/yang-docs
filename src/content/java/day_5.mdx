# Java 封装、多态、对象转型 笔记

---

## 一、封装（Encapsulation）

### 1. 核心理解

封装就是：

* 隐藏对象内部实现细节
* 只对外暴露必要的访问方式

简单理解就是：

**该藏起来的藏起来，该给外部用的再提供方法。**

---

### 2. 封装的作用

封装主要解决两个问题：

* **数据安全**
* **降低耦合**

---

### 3. 高内聚、低耦合

#### 高内聚

一个类尽量只做一类事情。

例如：

* `UserService` 只处理用户相关逻辑
* 不要同时又去写短信发送、文件上传等不相关逻辑

#### 低耦合

一个类内部实现怎么改，尽量不要影响其他类。

例如：

* 成员变量用 `private`
* 外部只能通过方法访问
* 以后内部实现改了，外部调用代码通常不用改

---

### 4. 权限控制的实际意义

| 权限          | 常见场景        |
| ----------- | ----------- |
| `private`   | 成员变量、内部实现细节 |
| 默认（default） | 同包内部协作类     |
| `protected` | 给子类使用的方法或属性 |
| `public`    | 对外公开的接口方法   |

---

### 5. JavaBean 规范

JavaBean 一般要求：

1. 类通常是 `public`
2. 提供无参构造器
3. 属性用 `private` 修饰
4. 提供对应的 getter / setter 方法

```java id="rnw1zb"
public class Person {
    private String name;
    private int age;

    public Person() {
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}
```

---

### 6. boolean 属性的命名

如果属性是 `boolean` 类型，getter 常写成：

```java id="oanvp4"
private boolean active;

public boolean isActive() {
    return active;
}
```

通常用 `isXxx()`，而不是 `getXxx()`。

---

## 二、多态（Polymorphism）

### 1. 核心理解

多态可以总结成一句话：

**编译时看左边，运行时看右边。**

意思是：

* 编译时，看引用类型中有没有这个方法
* 运行时，真正执行哪个方法，看对象实际类型

---

### 2. 多态的三个前提

多态成立必须同时满足三个条件：

1. 有继承关系，或者实现接口
2. 子类重写父类方法
3. 父类引用指向子类对象

---

### 3. 标准示例

```java id="xg5njy"
class Shape {
    public double calc() {
        return 0;
    }
}

class Square extends Shape {
    private int width;

    public Square(int width) {
        this.width = width;
    }

    @Override
    public double calc() {
        return this.width * this.width;
    }
}

class Circle extends Shape {
    private int radius;

    public Circle(int radius) {
        this.radius = radius;
    }

    @Override
    public double calc() {
        return this.radius * this.radius * 3.14;
    }
}
```

测试：

```java id="jbn44n"
public class ShapeCalc {
    public static void main(String[] args) {
        Shape circle = new Circle(3);
        Shape square = new Square(3);

        System.out.println(circle.calc()); // 28.26
        System.out.println(square.calc()); // 9.0
    }
}
```

---

### 4. 为什么这叫多态

因为：

```java id="ppngm8"
Shape s1 = new Circle(3);
Shape s2 = new Square(3);
```

虽然引用类型都是 `Shape`，但调用 `calc()` 时：

* `s1` 执行的是 `Circle` 的 `calc()`
* `s2` 执行的是 `Square` 的 `calc()`

同一个方法调用，因为对象不同，表现不同，这就是多态。

---

### 5. 多态的常见应用

多态很适合统一处理一组有共同父类的对象。

例如放进数组里：

```java id="mjlwm0"
Shape[] shapes = {new Circle(2), new Square(4)};

for (Shape s : shapes) {
    System.out.println(s.calc());
}
```

这样写的好处是：

* 调用方式统一
* 扩展新子类时，原有代码往往不用大改

---

## 三、多态中的方法调用规则

### 1. 成员方法

对于被重写的方法：

* 编译看左边
* 运行看右边

```java id="p6sjah"
Shape s = new Circle(3);
s.calc(); // 调用 Circle 中重写后的方法
```

---

### 2. 成员变量

成员变量**没有多态性**，看的是引用类型。

这个点只需要知道，不用展开太深。

---

### 3. 哪些方法不是动态绑定

下面这些方法一般不是多态调用：

* `static`
* `final`
* `private`
* 构造器

因为它们在编译阶段就基本确定了。

---

## 四、对象转型（Type Casting）

对象转型分两种：

* 向上转型
* 向下转型

---

## 五、向上转型（Upcasting）

### 1. 定义

子类对象赋值给父类引用。

```java id="jpwa34"
Circle c = new Circle(3);
Shape s = c;
```

也可以直接写：

```java id="kdjlwm"
Shape s = new Circle(3);
```

---

### 2. 特点

* 自动完成，不需要强制转换
* 安全
* 是多态的基础

---

### 3. 代价

向上转型后，只能通过父类引用访问父类中定义的方法和属性。

也就是说：

* 子类特有方法不能直接调用

```java id="vviyrn"
Shape s = new Circle(3);
// s.someCircleMethod(); // 如果是 Circle 特有方法，就不能调用
```

---

## 六、向下转型（Downcasting）

### 1. 定义

把父类引用再转回子类引用。

```java id="fhv0vb"
Shape s = new Circle(3);
Circle c = (Circle) s;
```

---

### 2. 特点

* 必须强制转换
* 有风险
* 前提是这个父类引用本来就指向子类对象

---

### 3. 错误示例

```java id="b0e0gb"
Shape s = new Shape();
Circle c = (Circle) s; // 报错
```

原因：

* `s` 实际指向的是 `Shape` 对象
* 它根本不是 `Circle`
* 所以运行时会抛出 `ClassCastException`

---

### 4. 安全写法

向下转型前一般先用 `instanceof` 判断：

```java id="v4r9ga"
Shape s = new Circle(3);

if (s instanceof Circle) {
    Circle c = (Circle) s;
    System.out.println(c.calc());
}
```

这样更安全。

---

### 5. JDK 16+ 的写法

如果是较新的 Java 版本，也可以写成：

```java id="vbl5xh"
if (s instanceof Circle c) {
    System.out.println(c.calc());
}
```

这个知道即可。

---

## 七、转型总结

| 转型方向 | 写法                        | 是否安全 | 说明                 |
| ---- | ------------------------- | ---- | ------------------ |
| 向上转型 | `Parent p = new Child();` | 安全   | 自动完成               |
| 向下转型 | `Child c = (Child) p;`    | 有风险  | 先做 `instanceof` 判断 |

---

## 八、封装和多态这一部分最该记住的点

### 封装

* 隐藏细节，对外暴露方法
* 成员变量一般用 `private`
* 通过 getter / setter 控制访问
* 封装的目的：安全、低耦合

### JavaBean

* 类通常是 `public`
* 有无参构造器
* 属性私有化
* 提供 getter / setter

### 多态

* 前提：继承、重写、父类引用指向子类对象
* 编译看左边，运行看右边
* 多态提高扩展性

### 转型

* 向上转型安全，自动完成
* 向下转型有风险，先用 `instanceof`
