# Java 面向对象之抽象（抽象类、接口）笔记

---

## 一、什么是抽象

### 1. 核心思想

抽象就是：

* 抓住事物共同的、本质的特征
* 忽略具体细节

在 Java 里可以理解为：

* 对外只关心“**能做什么**”
* 不关心“**具体怎么做**”

---

### 2. 抽象的作用

抽象主要有几个作用：

* 降低系统复杂度
* 提高代码可维护性
* 方便实现多态
* 统一一类对象的行为规范

---

### 3. 例子理解

例如图形：

* 圆形
* 矩形
* 三角形

它们的面积公式都不同，但都可以抽象出一个共同能力：

```java id="9nmj36"
求面积
```

调用者只需要知道“它能求面积”，不需要关心具体公式。

---

## 二、抽象方法（abstract method）

### 1. 定义

抽象方法只有方法声明，没有方法体。

```java id="8k7g7k"
public abstract double getArea();
```

---

### 2. 特点

* 必须用 `abstract` 修饰
* 没有方法体
* 以分号 `;` 结束
* 只能出现在**抽象类**或**接口**中

---

### 3. 注意点

抽象方法不能是：

* `private`
* `final`
* `static`

因为：

* `private`：子类看不到，没法实现
* `final`：不能重写，而抽象方法本来就要子类实现
* `static`：静态方法不依赖对象，不适合抽象成“重写”的形式

基础阶段记住前两条就够了。

---

### 4. 示例

```java id="91q8cm"
public abstract class Shape {
    public abstract double getArea();

    public abstract void draw(String color);
}
```

---

## 三、抽象类（Abstract Class）

### 1. 定义

用 `abstract` 修饰的类叫抽象类。

```java id="n2cqkq"
public abstract class Animal {
}
```

---

### 2. 抽象类的特点

| 特点      | 说明         |
| ------- | ---------- |
| 不能实例化   | 不能直接 `new` |
| 可以有抽象方法 | 交给子类实现     |
| 可以有普通方法 | 方便代码复用     |
| 可以有成员变量 | 和普通类一样     |
| 可以有构造器  | 供子类初始化使用   |

---

### 3. 为什么抽象类不能 new

因为抽象类本身通常表示一种“模板”或“不完整的类”，其中可能有抽象方法还没实现，所以不能直接创建对象。

---

### 4. 示例

```java id="tl2h1o"
public abstract class Animal {
    private String name;

    public Animal(String name) {
        this.name = name;
    }

    public void sleep() {
        System.out.println(name + " is sleeping.");
    }

    public abstract void makeSound();

    public String getName() {
        return name;
    }
}
```

子类实现：

```java id="8zzh3z"
public class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    @Override
    public void makeSound() {
        System.out.println(getName() + " says: Woof!");
    }
}
```

使用：

```java id="n2c6zw"
Animal dog = new Dog("Buddy");
dog.makeSound();
dog.sleep();
```

---

### 5. 子类和抽象类的关系

如果子类继承抽象类：

* 要么实现父类所有抽象方法
* 要么子类自己也声明成抽象类

例如：

```java id="gaqmpq"
abstract class A {
    abstract void test();
}

class B extends A {
    @Override
    void test() {
        System.out.println("实现了");
    }
}
```

## 四、接口（Interface）

### 1. 定义

接口是一种规范，表示某个类“**能做什么**”。

```java id="dlfumi"
public interface Flyable {
    void fly();
}
```

---

### 2. 接口的核心理解

接口主要不是用来放共性数据的，而是用来定义一组行为规范。

例如：

* 会飞：`Flyable`
* 会游泳：`Swimmable`

某个类只要实现这个接口，就必须提供对应行为。

---

### 3. 接口的特点

| 特点         | 说明           |
| ---------- | ------------ |
| 不能实例化      | 不能直接 `new`   |
| 类可以实现多个接口  | 弥补单继承限制      |
| 接口没有普通构造器  | 因为接口本身不能创建对象 |
| 主要用于定义行为规范 | 强调“能做什么”     |

---

## 五、接口中的成员

---

### 1. 常量

接口中的成员变量默认都是：

```java id="s57lik"
public static final
```

例如：

```java id="ptaz67"
interface Flyable {
    int MAX_SPEED = 1200;
}
```

实际上等价于：

```java id="yz3b1f"
public static final int MAX_SPEED = 1200;
```

---

### 2. 抽象方法

接口中的抽象方法默认都是：

```java id="81q37k"
public abstract
```

例如：

```java id="3h8a4y"
interface Flyable {
    void fly();
}
```

等价于：

```java id="2sp0cd"
public abstract void fly();
```

---

### 3. default 方法（JDK 8+）

接口中可以写带默认实现的方法。

```java id="e8ae4n"
interface Flyable {
    default void takeOff() {
        System.out.println("Preparing for takeoff...");
    }
}
```

#### 作用

主要是为了接口升级时，减少对旧实现类的影响。

---

### 4. static 方法（JDK 8+）

接口中可以有静态方法。

```java id="pfe6ee"
interface Flyable {
    static void showMaxSpeed() {
        System.out.println("Max speed is 1200");
    }
}
```

调用时要用接口名：

```java id="p0q4mt"
Flyable.showMaxSpeed();
```

---

### 5. private 方法（JDK 9+）

接口中还可以有私有方法，通常用于给 `default` 方法复用代码。

这个点了解即可，基础阶段不用深挖。

---

## 六、接口的实现

### 1. 语法

类实现接口用 `implements`

```java id="g0je4n"
class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("bird can fly");
    }
}
```

---

### 2. 多实现

一个类可以实现多个接口：

```java id="g4wcqf"
class Airplane implements Flyable, Landable {
}
```

这也是接口非常重要的一个特点。

---

## 七、抽象类和接口的区别

| 对比项     | 抽象类              | 接口                          |
| ------- | ---------------- | --------------------------- |
| 关键字     | `abstract class` | `interface`                 |
| 使用方式    | `extends`        | `implements`                |
| 继承/实现数量 | 只能继承一个           | 可以实现多个                      |
| 构造器     | 有                | 没有                          |
| 成员变量    | 可以有普通变量          | 只能是常量                       |
| 方法      | 可有抽象方法和普通方法      | 抽象方法、default、static、private |
| 设计作用    | 抽取共性、代码复用        | 定义规范、行为约束                   |

---

## 八、怎么理解“抽象类”和“接口”

### 1. 抽象类更像“模板”

它更适合描述：

* 一类对象的共同属性
* 一部分通用实现

例如：

* 动物都有名字
* 动物都会睡觉
* 但叫声不同

这时适合用抽象类。

---

### 2. 接口更像“能力 / 规范”

它更适合描述：

* 某个对象是否具有某种能力
* 不同类之间的统一行为规范

例如：

* 会飞
* 会跑
* 会游泳

这时适合用接口。

---

### 3. 简单记忆

* 抽象类：更偏“**是什么**”
* 接口：更偏“**能做什么**”

---

## 九、接口和抽象类可以配合使用

实际开发里经常是：

* **接口** 定义规范
* **抽象类** 提供公共实现
* **具体类** 真正完成功能

例如：

```java id="kcjlwm"
interface Measurable {
    double getArea();

    default void printArea() {
        System.out.println("Area: " + getArea());
    }
}
```

抽象类：

```java id="gwhe4b"
abstract class Shape implements Measurable {
    protected String shapeName;

    public Shape(String shapeName) {
        this.shapeName = shapeName;
    }

    public void displayName() {
        System.out.println("Shape: " + shapeName);
    }
}
```

具体类：

```java id="jlwmby"
class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        super("Circle");
        this.radius = radius;
    }

    @Override
    public double getArea() {
        return Math.PI * radius * radius;
    }
}
```

## 十、最佳实践

### 1. 什么时候用抽象类

当多个子类之间：

* 有共同属性
* 有共同方法实现
* 只是某些行为不同

这时优先考虑抽象类。

---

### 2. 什么时候用接口

当你只是想定义一组规范，强调“实现这个功能”，这时优先用接口。

特别是：

* 需要多实现
* 不同类层次都可能拥有这个能力

更适合用接口。

---

### 3. 实际开发建议

* 行为规范优先考虑接口
* 代码复用再考虑抽象类
* 两者可以组合使用

---

## 十一、这一部分最该记住的点

### 抽象

* 抽象就是提取共同点，忽略细节
* 目的是降低复杂度、方便扩展

### 抽象方法

* 只有声明，没有方法体
* 必须用 `abstract`
* 只能出现在抽象类或接口中

### 抽象类

* 不能直接实例化
* 可以有抽象方法，也可以有普通方法
* 可以有构造器和成员变量
* 子类要实现所有抽象方法，否则子类也要是抽象类

### 接口

* 用来定义规范
* 类可以实现多个接口
* 接口中变量默认是常量
* 接口中抽象方法默认是 `public abstract`

### 抽象类 vs 接口

* 抽象类：偏共性、代码复用
* 接口：偏规范、能力约束
