## 常量

### `final` 关键字定义常量
- **基础用法**：`final int MAX_VALUE = 100;`
- **不可变性细节**：
    - **基本类型**：数值不可变。
    - **引用类型**：**引用地址不可变**，但对象内部的属性是可以修改的。
        ```java
        final User u = new User("张三");
        u.setName("李四"); // 允许，对象内容变了
        // u = new User("王五"); // 报错，引用指向不能变
        ```
- **命名规范**：常量名通常全部大写，单词间用下划线分隔（`MAX_SPEED`）。

### 常量的分类与优化
| 类型 | 代码示例 | 存储/编译特点 |
| :--- | :--- | :--- |
| **编译期常量** | `static final int A = 10;` | 编译时直接替换为字面量，不依赖类加载 |
| **运行时常量** | `static final int B = "hello".length();` | 类初始化时才确定值 |

## static

### 核心理解：属于类，而非对象
- **生命周期**：随类的加载而初始化，随类的卸载而消亡（贯穿整个程序运行周期）。
- **共享性**：所有该类的实例共享同一个 `static` 变量副本。

### 静态方法调用的限制（原理深入）
- **为什么 static 不能调用非 static？**
    - 静态方法属于类，没有 `this` 指针隐式传递。
    - 非静态方法依赖具体的对象实例才能访问属性。在没有对象存在时，JVM 找不到操作的目标数据。

### 静态代码块（很重要，并非“没啥屌用”）
它通常用于**加载本地动态链接库**（JDBC驱动）或**初始化复杂的静态集合**。

```java
class DBHelper {
    static Properties props = new Properties();
    
    static {
        // 类加载时读取配置文件，只执行一次，确保资源准备就绪
        try (InputStream in = DBHelper.class.getResourceAsStream("/db.properties")) {
            props.load(in);
        } catch (IOException e) {
            throw new RuntimeException("初始化失败");
        }
    }
}
```

### 静态导入（JDK 1.5+ 补充）
```java
import static java.lang.Math.*; // 导入Math类的所有静态方法

// 使用时直接写方法名，无需类名
double r = cos(PI * 2);
```

## 简单内存模型（JVM 内存区域细化）

您笔记中的概念对应的是 **JVM 运行时数据区**。这里进行精准修正：

### 1. 栈（Stack）—— 方法运行的“操作台”
- **结构**：**线程私有**，先进后出（LIFO）。
- **存储内容**：**栈帧（Stack Frame）**。
    - **局部变量表**：存放基本类型数值、引用类型的地址（指针）。
    - **操作数栈**：计算过程的临时数据中转站。
- **重点纠正**：您笔记里写“常量”在栈中。**常量（字符串常量、static final常量）不在栈中**，它们在**方法区（元空间）**或**堆中的常量池**。

### 2. 堆（Heap）—— 对象的“仓库”
- **存储内容**：**所有的对象实例**（`new` 出来的东西）和**数组**。
- **结构**：无序，由 GC（垃圾回收器）管理。
- **引用示意图**：
    ```text
    栈内存                      堆内存
    User u  -----> [ id=1, name引用 ---> [ "张三" (字符串常量池) ] ]
    ```

### 3. 方法区（Method Area）—— 类的“蓝图/档案馆”
- **存储内容**：类的结构信息（字段、方法字节码）、**运行时常量池**、静态变量。
- **演变历史（重要修正）**：
    - **JDK 7 及之前**：叫永久代（PermGen），字符串常量池在方法区。
    - **JDK 8 及之后**：**彻底废弃永久代**，改用本地内存实现的**元空间（Metaspace）**。
    - **字符串常量池位置变化**：JDK 7 开始**字符串常量池移到了堆内存**。
- **溢出报错**：
    - 永久代溢出：`java.lang.OutOfMemoryError: PermGen space`
    - 元空间溢出：`java.lang.OutOfMemoryError: Metaspace`

## this

### 本质定义
> 指向**当前对象**的引用。在对象创建之前，`this` 是不存在的。

### 四种使用场景深度解析

#### 1. 构造器中：指向正在初始化的对象
```java
public Person(String name) {
    this.name = name; // 区分入参 name 和成员变量 name
}
```

#### 2. 普通方法中：指向调用该方法的对象
```java
public void call() {
    System.out.println(this); // 打印当前对象的内存地址标识
}
```

#### 3. 静态上下文的禁区
- **原因**：静态方法调用时，可能连对象都还没创建，`this` 无法指向任何东西。

#### 4. `this()` 构造器调用
- **语法**：`this(参数);`
- **限制**：**必须是构造器代码块的第一条语句**。
- **作用**：减少代码冗余，复用构造器逻辑。

```java
class Car {
    int speed;
    String color;

    Car() {
        this(100, "Red"); // 调用双参构造器，必须首行
    }

    Car(int speed, String color) {
        this.speed = speed;
        this.color = color;
    }
}
```

### `this` 逃逸问题（进阶概念补充）
在构造器中，如果 `this` 还未完全初始化就被传递给其他线程（比如构造器内启动新线程调用该对象的方法），可能会因指令重排序导致看见对象尚未完全初始化好的状态。**最佳实践**：不要在构造器中启动线程或发布 `this` 引用。
