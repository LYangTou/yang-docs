# Java 常量、static、简单内存模型、this 笔记

---

## 一、常量

### 1. `final` 定义常量

使用 `final` 修饰变量，表示这个变量**只能赋值一次**。

```java
final int MAX_VALUE = 100;
```

---

### 2. `final` 修饰不同类型的区别

#### 基本类型

值不能变。

```java
final int a = 10;
// a = 20; // 报错
```

#### 引用类型

**引用地址不能变**，但对象内部内容可以变。

```java
final User u = new User("张三");
u.setName("李四"); // 允许，改的是对象内容
// u = new User("王五"); // 报错，引用不能重新指向别的对象
```

---

### 3. 命名规范

常量名一般：

* 全部大写
* 单词之间用下划线分隔

例如：

```java
final int MAX_SPEED = 120;
```

---

### 4. 常量分类

| 类型    | 示例                                     | 特点       |
| ----- | -------------------------------------- | -------- |
| 编译期常量 | `static final int A = 10;`             | 编译时可直接确定 |
| 运行时常量 | `static final int B = "abc".length();` | 运行时才能确定  |

#### 理解

* **编译期常量**：值在编译阶段就已经确定
* **运行时常量**：虽然也是 `final`，但值要等程序运行时才能得到

---

## 二、static

### 1. 核心理解

`static` 表示“**属于类**”，不属于某个具体对象。

---

### 2. 静态变量

* 属于类
* 所有对象共享一份

```java
class Student {
    static String school = "ABC";
}
```

调用时一般推荐：

```java
System.out.println(Student.school);
```

---

### 3. 静态方法

* 属于类
* 可以通过类名直接调用

```java
class MathUtil {
    public static int add(int a, int b) {
        return a + b;
    }
}
```

调用：

```java
MathUtil.add(1, 2);
```

---

### 4. 为什么静态方法不能直接访问非静态成员

因为：

* 静态方法属于类
* 非静态成员属于对象
* 调用静态方法时，对象可能还没有创建

所以静态方法里不能直接写：

```java
this.name
```

也不能直接访问普通成员变量和普通方法。

---

### 5. 静态代码块

* 类加载时执行
* 只执行一次
* 常用于初始化静态资源

```java
class DBHelper {
    static Properties props = new Properties();

    static {
        System.out.println("类加载时执行一次");
    }
}
```

#### 作用

常见用途：

* 读取配置文件
* 初始化静态集合
* 加载驱动或资源

---

### 6. 静态导入

可以直接导入某个类的静态成员，使用时不用再写类名。

```java
import static java.lang.Math.*;

double r = cos(PI * 2);
```

## 三、简单内存模型

这里按基础学习的角度记住三个区域就够了：

---

### 1. 栈（Stack）

栈主要用于**方法运行**。

里面一般存放：

* 局部变量
* 方法调用过程中的临时数据
* 引用变量本身

例如：

```java
User u = new User();
```

这里：

* `u` 这个引用变量在栈中
* `new User()` 创建出来的对象在堆中

---

### 2. 堆（Heap）

堆中主要存放：

* 对象
* 数组

```java
new User();
new int[10];
```

这些都在堆里。

#### 特点

* 堆由垃圾回收器（GC）管理
* `new` 出来的内容基本都在堆中

---

### 3. 方法区（Method Area）

方法区主要存放类相关信息，例如：

* 类的信息
* 方法信息
* 静态变量
* 运行时常量池

---

### 4. 关于 JDK 8 之后

你这部分原笔记方向是对的，整理后记这一句就够：

* JDK 8 以后，永久代被移除，改成了 **元空间（Metaspace）**
* 字符串常量池在 JDK 7 开始已经移到堆中

这个知道即可，不需要往下挖太深。

---

### 5. 最简单的理解图

```text
栈：局部变量、引用
堆：对象、数组
方法区：类信息、静态变量、常量池
```

## 四、this

### 1. 本质

`this` 表示**当前对象的引用**。

可以简单理解成：

* 谁在调用这个方法
* `this` 就指向谁

---

### 2. 在构造器中使用

最常见用途：区分成员变量和局部变量。

```java
class Person {
    String name;

    public Person(String name) {
        this.name = name;
    }
}
```

这里：

* `name` 是构造器参数
* `this.name` 是当前对象的成员变量

---

### 3. 在普通方法中使用

表示调用当前方法的对象。

```java
class Person {
    public void show() {
        System.out.println(this);
    }
}
```

---

### 4. 在静态方法中不能使用 `this`

因为：

* `this` 依赖对象存在
* 静态方法属于类
* 调用静态方法时可能根本没有对象

所以静态方法中不能写：

```java
this.name
```

---

### 5. `this()` 调用本类构造器

可以在一个构造器中调用同类的另一个构造器。

```java
class Car {
    int speed;
    String color;

    Car() {
        this(100, "Red");
    }

    Car(int speed, String color) {
        this.speed = speed;
        this.color = color;
    }
}
```

#### 注意

`this()`：

* 只能在构造器里使用
* **必须写在第一行**

---

## 五、补充：this 和 this()

这两个很容易混：

### `this`

表示当前对象

```java
this.name = name;
```

### `this()`

表示调用本类其他构造器

```java
this(100, "Red");
```

## 六、这一部分最该记住的点

### 常量

* `final` 修饰变量，表示只能赋值一次
* 基本类型不能改值
* 引用类型不能改指向，但对象内容可能能改
* 常量名一般全大写

### static

* `static` 属于类，不属于对象
* 静态变量所有对象共享
* 静态方法不能直接访问非静态成员
* 静态代码块在类加载时执行一次

### 内存模型

* 栈：局部变量、引用
* 堆：对象、数组
* 方法区：类信息、静态变量、常量池

### this

* `this` 表示当前对象
* `this.name` 常用于区分成员变量和局部变量
* `this()` 用于调用本类其他构造器
* `this()` 必须写在构造器第一行
