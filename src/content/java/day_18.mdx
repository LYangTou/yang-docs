# 注解与反射

## 元数据Metadata
- 描述数据的数据，或者说是数据的附加信息
- 再软件开发中，元数据通常用于描述程序的配置信息、注解信息、文档信息
- 元数据可以分为三个层次
  - 描述数据的属性和结构，比如数据类型、字段名、长度、精度等
  - 描述数据的来源和格式，比如数据源、数据格式、数据版本等
  - 描述数据的用途和访问权限，比如数据的许可证、数据的保密性、数据的可用性等

比如 `String str`， str是数据，规定str多长就是元数据。

## 注解
- 是Java一种特殊语法元素
- 提供了一种在程序中添加元数据（Metadata）的方法
- 可以用于代码的编译、运行和解析时期，为程序的运行提供一些额外的信息

方法的重写`@Override`就是一种注解。其实和注释有类似的方向，也就是都提供一些信息

### 注解的本质和分类
- 用法： @注解名
- 本质：特殊Java接口 `@interface`
- 分类
  - 运行机制：源码注解，编译时注解，运行时注解
  - 代码来源：JDK注解，第三方注解，自定义注解
  - 特殊注解：元注解

#### JDK内置注解
- @Override：重写类的方法
- @Deprecated： 表示类、方法、接口已废弃
- @SuppressWarings：抑制编辑器的警告，可以传一个value
- @FunctionInterface：函数式接口
- @SafeVarags

#### JDK元注解
- @Retention: 注解的保留策略，表明的是表留的阶段
  - RetentionPolicy
    - SOURCE
    - CLASS: 默认行为
    - RUNTIME：只有注解保留策略选择这个得时候，才能进行反射
- @Target: 注解的作用目标，指定注解用于修饰那些程序元素（就是限定用在哪，比如Override只能用在方法上）
  - ElementType
- @Inherited： 指定注解具有继承性
- @Documented：注解将被包含在Javadoc中
- @Repeatable：可重复注解
- @Native：修饰成员变量，可被本地代码引用（不常用）

#### 自定义注解
- 用@interface定义注解接口，并给出注解的名称以及元注解
- 在注解接口中定义注解参数。注解参数的定义方式和接口方法类似，但是没有方法体，而且可以指定默认值。
    ```java
    @Retention(RetentionPolicy.RUNTIME)
    @Target({ElementType.TYPE, ElementType.PARAMETER})
    public @interface SingleParameterAnnotation {
        String value() default "unknown";
        String name() default "aga";
    }
    ```
- 在程序中使用注解。可以在类、方法、字段等元素上使用注解，以便进行元数据描述和处理。

Repeatable使用
```java

@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface Animals {
    Animal[] value();
}


@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
@Repeatable(Animals.class)
public @interface Animal {
    String value();
}

@Animal("a")
@Animal("b")
@Animal("c")
public class AnnotationDemo {

}
```

## 反射
比如 要调用一个类的方法，只知道路径名，但是不知道具体实现，应该怎么办？
比如 没有为私有属性提供`getter/setter`，非要修改私有属性字段怎么办？

Java 反射的机制： 允许我们再运行时获取类的信息，并动态地创建对象、访问修改对象属性和调用方法。这种灵活性使得Java程序可以更加动态和可拓展。

反射机制的主要作用：
- 动态创建对象：可以动态地加载类，并创建他的对象。这对于创建灵活的框架和库非常有用。
- 动态获取类的信息：例如类名、父类、接口、字段和方法等。可以在运行时获取类的信息，并根据这些信息执行操作
- 动态调用方法：包括私有方法
- 动态修改属性：包括私有属性。使得我们可以在运行时修改对象的状态 （很少用）
- 反射重要应用：动态代理

Java反射机制重要核心类`Class`类，用来描述类的信息。

### 如何获取Class对象
- 使用对象的getClass()方法获取Class对象
- 使用.class获取Class对象
- 使用Class.forName()方法获取Class对象
- （了解）使用类加载器ClassLoader获取Class对象
- 特殊用法：基本类型包装类的Type
- 注意：Class对象是在编译期间生成的，所以每个类在JVM中只有一个Class对象。
```java
  // 方法1 .class
        Class<String> clazz1 = String.class;
        System.out.println(clazz1);
        Class<MyHashMap> clazz2 = MyHashMap.class;
        System.out.println(clazz2);

        // 方法2 获取对象的 getClass()
        String s = "hell world";
        Class<? extends String> clazz3 =  s.getClass();
        MyHashMap<Number, String> m = new MyHashMap();
        Class clazz4 = m.getClass();

        // 方法3： Class.forName()
        try {
            Class clazz5 = Class.forName("java.lang.String");
            System.out.println(clazz5);
            Class clazz6 = Class.forName("MyHashMap");
            System.out.println(clazz6);
        } catch (ClassNotFoundException ex) {
            throw new RuntimeException(ex);
        }

        // (了解) 方法4： 利用classLoader
        ClassLoader classLoader = ObtainClassDemo.class.getClassLoader();
        try {
            Class clazzz = classLoader.loadClass("java.lang.String");
        } catch (ClassNotFoundException e) {
            throw new RuntimeException(e);
        }

        // 特殊获取Class对象（只能用于基本类型包装类）
        Class<Integer> i = Integer.TYPE;
```
### 动态创建对象

```java
public class ObjectCreationDemo {
    public static Object getInstance() {
        Random random = new Random();
        int seed = random.nextInt(2);
        String className = "";
        switch (seed){
            case 0:
                className = "java.lang.String";
                break;
            case 1:
                className = "MyHashMap";
                break;
            default:
                System.out.println("no");
        }

        try {
           Class clazz = Class.forName(className);
           return clazz.newInstance();
        } catch (ClassNotFoundException e) {
            throw new RuntimeException(e);
        } catch (InstantiationException e) {
            throw new RuntimeException(e);
        } catch (IllegalAccessException e) {
            throw new RuntimeException(e);
        }
    }
    public static void main(String[] args) {
        Class<MyHashMap> clazz = MyHashMap.class;
        try {
            // 1.需要无参构造器
            // 2.无参构造器需要有访问权限
            MyHashMap<Number, String> m1 = clazz.newInstance();
            System.out.println(m1);
        } catch (InstantiationException e) {
            throw new RuntimeException(e);
        } catch (IllegalAccessException e) {
            throw new RuntimeException(e);
        }

        for (int i = 0;i < 10;i++) {
            System.out.println(getInstance());
        }
    }
}
```

### 如何动态调用方法

#### 最简单：调用Public无参数无返回值方法
```java
// 1. 获取目标类的class对象
MyClass obj = new MyClass(10);
Class clazz = obj.getClass()

// 2. 获取目标方法的method对象
Method d = clazz.getMethod("doNothing");

// 3. 调用method对象的invoke方法
Object dr = doNothing.invoke(obj)
```

#### 调用public有参数有返回值犯法
```java
// 1. 获取目标类的class对象
MyClass obj = new MyClass(10);
Class clazz = obj.getClass()

// 2. 获取目标方法的method对象
Method d = clazz.getMethod("sum", int.class, String.class);

// 3. 调用method对象的invoke方法, 参数对应method对象
Object dr = doNothing.invoke(obj, 10， "a")

```

#### 调用private有参数有返回值方法

```java
// 1. 获取目标类的class对象
MyClass obj = new MyClass(10);
Class clazz = obj.getClass();
// 2. 获取目标方法的method对象
Method d = clazz.getDelclaredMethod("sum", String.class, String.class);
// 3. 调用private方法需要设置accessible属性
d.setAccessible(true);
// 4. 调用method对象的invoke方法, 参数对应method对象
Object dr = doNothing.invoke(obj, 10， "a");
```

#### 调用静态方法
```java
// 1. 获取目标类的class对象
MyClass obj = new MyClass(10);
Class clazz = obj.getClass();
// 2. 获取目标方法的method对象
Method d = clazz.getDelclaredMethod("sum", String.class);
// 3. 调用private方法需要设置accessible属性
d.setAccessible(true);
// 4. 调用method对象的invoke方法, 参数对应method对象, 把object设置为null
Object dr = doNothing.invoke(null, "a");

```

### 动态修改属性/动态调用构造器
- 对于private要用declared
- 对于private要把accessible设为true

```java
Person person = new Person()

Field ageFiled = Person.getClass().getDeclaredField("age")
// 将访问权限设置为true
ageFiled.setAccessible(true)
// set属性
ageFiled.set(person, 60)
```

```java
Class<Person> personClazz = Person.class;
// 获取对应要调用的构造器
Constructor constructor = personClazz.getDeclaredContructors(String.class);
// 设置可访问性为true
constructor.setAccessible(true)
// 利用newInstance初始化对象，注意参数需要和构造器传入参数对应
Person person = constructor.newInstance("af")
```
